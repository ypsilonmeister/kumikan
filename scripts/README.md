# Kanji Data Scripts

This directory contains helper data and scripts for expanding the kanji recipe dictionary.

## Source Files

- `kyoiku.csv`: Kyoiku kanji list fetched from `fnshr/kyo-kan`.
- `ids.txt`: IDS decomposition data fetched from `cjkvi/cjkvi-ids`.
- `data_sources.json`: Generated by `fetch_kanji_data.mjs`; records source repositories, pinned commit hashes, raw file paths, fetch time, and license metadata returned by GitHub.

Always use `data_sources.json` to identify the exact upstream revision behind generated recipe suggestions. If the raw files are regenerated, commit the updated `data_sources.json` together with them.

## Workflow

1. `node scripts/fetch_kanji_data.mjs`
   - Fetches `kyoiku.csv` and `ids.txt` from pinned GitHub commit hashes.
   - Writes `scripts/data_sources.json` with reproducibility metadata.

2. `node scripts/process_kanji.mjs`
   - Reads the raw data and current `src/domain/recipes.ts`.
   - Writes `recipes_suggested.json`, `recipes_suggested.md`, and `cannot_decompose.json`.
   - The generated suggestions are machine-extracted candidates, not a safe-to-add list.

3. `node scripts/filter-recipes.mjs`
   - Filters the Markdown suggestions into `accepted_recipes.txt` using the current project allowlist.
   - Treat this as a starting point for human review, especially for child-readable card parts.

## Review Notes

Do not bulk-add `recipes_suggested.md` directly. It can contain IDS fragments and uncommon glyph pieces that are technically valid Unicode but poor game cards. Prefer recipes whose parts are already readable kanji, existing game parts, or radicals with readings/images in `PART_READINGS`.
